Bookstore REST API Documentation

1. Introduction

The Bookstore REST API is a fictional API designed to simulate how a real-world bookstore management system functions. It enables developers to perform CRUD operations (Create, Read, Update, Delete) on books within the database. In addition to managing books, this API can be extended to include features for handling authors, purchases, and user authentication.

This documentation aims to provide a detailed and developer-friendly guide on how to use the Bookstore REST API effectively, including endpoint references, request examples, and response formats.

Base URL

Authentication

At this stage, the Bookstore API does not require authentication for basic operations. However, future versions may require an API key or OAuth2 authentication for protected endpoints like adding or deleting books.

2. Endpoints

Below is a list of the core endpoints available in the API:

  • GET /books - Retrieve all books
  • GET /books/:id - Retrieve a book by its ID
  • POST /books - Add a new book to the store
  • PUT /books/:id - Update an existing book
  • DELETE /books/:id - Remove a book from the store

Each endpoint is documented in detail below, including request parameters, expected responses, and example usage.

3. Get All Books

GET /books

Description: Retrieves a list of all books currently in the store.

Response Format:

Success Response:

  • Code: 200 OK
  • Content: JSON array of books

Error Handling:

  • Code: 500 Internal Server Error
  • Content: { "error": "Unable to fetch books" }

4. Get a Book by ID

GET /books/:id

Description: Retrieves the details of a specific book based on its unique ID.

URL Parameters:

  • id (integer): The ID of the book to retrieve

Success Response:

Error Responses:

7. Delete a Book

DELETE /books/:id

Description: Deletes a book from the bookstore by its ID.

URL Parameters:

  • id (integer): The ID of the book to delete

Success Response:

  • Code: 200 OK
  • Content: { "message": "Book deleted successfully" }

Error Responses:

  • Code: 404 Not Found
  • Content: { "error": "Book not found" }
  • Code: 500 Internal Server Error
  • Content: { "error": "Failed to delete book" }

8. Error Codes

The Bookstore API uses standard HTTP status codes to indicate the success or failure of an API request:

Code

Description

200

OK - The request was successful

201

Created - Resource was created

400

Bad Request - Invalid input

401

Unauthorized - Auth required

404

Not Found - Resource missing

500

Internal Server Error

Each error response is returned with a JSON object containing an error field with a human-readable message.
